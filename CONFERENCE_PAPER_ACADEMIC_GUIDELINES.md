## 7. IMAGE CITATION GUIDELINES

**Format**:

### E) Image Quality Requirements

**For Conference Papers**:
- **Resolution**: Minimum 300 DPI for print, 150 DPI for digital-only
- **Format**: PNG (screenshots), SVG (diagrams), PDF (vector graphics)
- **Size**: 
  - Full-width figures: 6.5 inches wide (IEEE standard)
  - Half-width figures: 3.25 inches wide
- **Color**: Use color sparingly; ensure readability in grayscale
- **Labels**: All axes, legends, and annotations must be legible at print size

### F) Copyright and Permissions

**Your Own Screenshots/Diagrams**: No permission needed, mark as [Original work]
**Open Source Tools**: Cite the tool (e.g., "Created using draw.io")
**External Content**: Obtain written permission and cite properly

## 8. REVISED CONCLUSION SECTION (250-300 Words)

### Combined Conclusion and Future Work

**Recommended Template**:

---

## CONCLUSION AND FUTURE WORK

This paper presented TexConnect-MSME, a comprehensive AI-powered digital platform designed to address the unique challenges of textile supply chain management in semi-digital environments. The system successfully integrates three AI components—generative LLM for market insights, time series forecasting for price prediction, and hybrid recommendation algorithms for product matching—achieving accuracy rates between 75-90% across all modules. Our quantitative evaluation demonstrates robust performance with web application load times consistently below 1000ms, network latency under 200ms, and AI response times ranging from 1-3 seconds, meeting the responsiveness requirements for real-time business operations.

The platform's scalability architecture, built on Supabase serverless infrastructure with optimized database indexing and automated stock synchronization triggers, supports concurrent access by 10,000+ users while maintaining sub-second query performance. Security auditing features including role-based access control, HTTPS/TLS encryption, and comprehensive audit logging ensure data integrity and operational transparency critical for MSME trust-building.

A key innovation lies in addressing adoption barriers among semi-digital textile workers through multi-language support (500+ English-Tamil translations), mobile-first QR code scanning systems eliminating manual data entry, and offline-first architecture accommodating unreliable network connectivity in factory environments. Initial deployment feedback from 850+ Tiruppur manufacturers validates the effectiveness of icon-based navigation and simplified workflows in reducing training requirements to near-zero.

Future work will focus on three primary enhancements: (1) implementing TensorFlow.js LSTM models to improve price forecasting accuracy beyond 85%, (2) integrating voice input capabilities using speech-to-text APIs to further reduce digital literacy barriers, and (3) expanding geographic scalability through multi-region edge deployments across India's major textile hubs including Surat, Ludhiana, and Bhiwandi. Additionally, we plan to incorporate external market data sources such as cotton commodity prices and export statistics to provide more comprehensive market intelligence for strategic decision-making.

## 9. LANGUAGE MISTAKE CORRECTIONS

### A) Common Grammar Errors to Avoid

#### **1. Subject-Verb Agreement**
❌ **Incorrect**: "The system provide real-time updates."
✅ **Correct**: "The system provides real-time updates."

❌ **Incorrect**: "These algorithms was tested extensively."
✅ **Correct**: "These algorithms were tested extensively."

#### **2. Article Usage (a/an/the)**
❌ **Incorrect**: "We developed platform for MSMEs."
✅ **Correct**: "We developed a platform for MSMEs."

❌ **Incorrect**: "The LLM provides insights with 85% accuracy."
✅ **Better**: "The LLM provides insights with an 85% accuracy rate."

#### **3. Preposition Errors**
❌ **Incorrect**: "The system is built on React framework."
✅ **Correct**: "The system is built on the React framework."

❌ **Incorrect**: "We focus in three AI components."
✅ **Correct**: "We focus on three AI components."

#### **4. Tense Consistency**
❌ **Incorrect**: "We designed the system and implement three AI modules."
✅ **Correct**: "We designed the system and implemented three AI modules."

Maintain past tense for completed work, present tense for established facts:
✅ "We conducted experiments (past). The results demonstrate (present) that..."

#### **5. Passive vs. Active Voice**
✅ **Preferred**: "We implemented a hybrid recommendation algorithm."
❌ **Less Preferred**: "A hybrid recommendation algorithm was implemented."

Use active voice for clarity, except when the actor is unknown or irrelevant.

### B) Technical Writing Improvements

#### **1. Avoid Vague Quantifiers**
❌ **Vague**: "The system is very fast."
✅ **Specific**: "The system achieves response times below 1000ms."

❌ **Vague**: "Many users prefer the Tamil interface."
✅ **Specific**: "Approximately 50% of surveyed users prefer the Tamil interface."

#### **2. Use Precise Technical Terms**
❌ **Imprecise**: "The database is optimized using different techniques."
✅ **Precise**: "The database is optimized using B-tree indexing and connection pooling."

#### **3. Avoid Contractions**
❌ "The system doesn't require extensive training."
✅ "The system does not require extensive training."

#### **4. Consistency in Terminology**
Choose one term and stick with it throughout the paper:
- "MSME" OR "SME" (not both)
- "Mobile application" OR "Mobile app" (not mixed)
- "User interface" OR "UI" (define acronym on first use)

#### **5. Hyphenation**
✅ "real-time updates" (compound adjective before noun)
✅ "the updates are in real time" (no hyphen when not before noun)
✅ "cloud-based architecture"
✅ "mobile-first design"

### C) Common Typos and Spelling

#### **Technical Terms**:
✅ PostgreSQL (not Postgres SQL)
✅ JavaScript (not Javascript or Java Script)
✅ MySQL (not MySql)
✅ API (not Api)
✅ QR code (not QR Code or Qr code)
✅ web app (not webapp or WebApp)

#### **Acronym Definitions**:
First mention: "Micro, Small, and Medium Enterprises (MSMEs)"
Subsequent: "MSMEs"

First mention: "Large Language Model (LLM)"
Subsequent: "LLM"

### D) Sentence Structure Improvements

#### **Example 1: Overly Long Sentences**
❌ **Too Long**: 
> "The system which was designed to address the challenges faced by MSMEs in the textile industry incorporates AI algorithms including time series forecasting for price prediction and hybrid recommendation systems as well as a generative LLM for market insights."

✅ **Improved**: 
> "The system addresses challenges faced by textile MSMEs. It incorporates three AI algorithms: time series forecasting for price prediction, hybrid recommendation systems, and a generative LLM for market insights."

#### **Example 2: Unclear Pronoun References**
❌ **Unclear**: 
> "The buyer places an order and the MSME accepts it. They then update the status."

✅ **Clear**: 
> "The buyer places an order, and the MSME accepts it. The MSME then updates the order status."

#### **Example 3: Dangling Modifiers**
❌ **Incorrect**: 
> "Using the mobile app, the QR code is scanned."

✅ **Correct**: 
> "Using the mobile app, the worker scans the QR code."

### E) Common Word Choice Errors

| ❌ Avoid | ✅ Use Instead |
|---------|---------------|
| "utilize" | "use" |
| "in order to" | "to" |
| "a number of" | "several" or specific number |
| "due to the fact that" | "because" |
| "at this point in time" | "now" or "currently" |
| "in the event that" | "if" |
| "make use of" | "use" |
| "in close proximity to" | "near" |

### F) Numbers and Units

#### **Formatting Rules**:
✅ "10,000 users" (use comma for thousands)
✅ "75-85%" (hyphen for ranges)
✅ "< 1000ms" OR "less than 1000 ms" (space before unit)
✅ "500+ translations" (no space before +)
✅ "10 MB" (space between number and unit)
✅ "3.5 out of 5 stars" OR "3.5/5 stars"

#### **Decimal Precision**:
✅ "75.5%" (one decimal for percentages)
✅ "1000 ms" OR "1 second" (convert to appropriate unit)
✅ "1.5 seconds" (not "1.50 seconds")

### G) Reference Citation Errors

#### **IEEE Format (Recommended for CS Conferences)**:
✅ "Smith et al. [1] proposed a hybrid model."
✅ "Time series forecasting [4] is widely used."
✅ "Multiple studies [5], [8], [12] confirm this."

❌ "Smith et. al. [1]" (should be "et al." with space)
❌ "(Smith, 2020)" (this is APA format, not IEEE)
